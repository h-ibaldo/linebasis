# Constraint
All commit messages must strictly follow the Conventional Commits specification.

# Rationale
To enable automated release note generation, semantic versioning, and a readable project history.

# Rules
- **Format**: `<type>(<scope>): <description>`
- **Types**: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `perf`.
- **Header**: Max 72 characters.
- **Mood**: Imperative (e.g., "add" not "added").
- **Body**: Optional, but required for breaking changes.
